Bachelor's Thesis

Title: "Digital Twin for the programming, the simulation, and the control of a Robot."
Author: Alessio Maria Falchi
Supervisor: Prof. Luca Iocchi

Overview

Bachelor's thesis project focused on showing the capabilities of a digital twin for robot programming and control. The project demonstrates how simulator-based development can enhance robotics programming workflow.

The development environment has been containerized with Docker to enhance portability. The image is based on Ubuntu Noble Numbat, with a ROS2 Jazzy installation. Gazebo Harmonic has been used for simulating the robot behaviors. All the tests have been performed on the virtual model of the SMARRtino educational robot, based on the MARRtino project and provided by Robotics 3D.

Content Details

Introduction
The main concepts are introduced, from the definition of a robot to the fundamental techniques used for its programming.

Development environment
Further details about the used software tools are provided, along with the chosen configuration and conventions.

Open-Loop Control
After a brief introduction to the main control theory concepts (with focus on the open-loop technique), the results of several motion control functions are shown. This chapter aims to show the ease of implementation of an open-loop controller, along with its limitations. The possibility to visualize data and plots resulting from each simulation highlights the potentialities of the offline robot programming technique.

Closed-Loop Control
In order to compensate the open-loop control limitations, the closed-loop one is introduced. Other simulations are accordingly done, aiming for further precision. Results are shown and compared with the ones previously obtained from the open-loop functions.

Conclusion
The advantages of using a simulated environment are repeated: being able to easily visualize the robot behaviors and the related resulting data proves to be the winning point of the offline programming technique. Further tools and potentialities provided by the simulator are mentioned, showing how could they be a valid starting point for solving more complex control problems.
